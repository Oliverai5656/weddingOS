# Task 02 — 主页 / 今天视图

> 依赖：Task 00, 01
> 这是**招牌页面**，也是用户每天打开 App 的理由。它的工作是制造平静，不是展示信息。

---

## 目标
一眼让用户知道：还有多久、接下来做什么、整体进展如何——而且感觉平静、被鼓励。

---

## 范围（要做）

### 页面内容（按重要性）
1. **倒计时** —— "还有 47 天"。陈述式，不尖叫。无红色、无激进动画。
2. **接下来要做的事** —— **最多 3 项**。从清单里取最紧急的 now 任务。主要操作 = 勾选 / 进入处理。
3. **温和的进度** —— 完成正向措辞（"这周完成了 3 件事 🎉" / "你们一起完成了 30%"）。
   - **绝不**亏欠式（"还剩 12 项" / "进度落后"）。
   - **不要进度条压力、不要游戏化。**
4. **伴侣动态（轻量）** —— 如有，显示对方最近的动作（"Sarah 确认了场地"）。强化"一起做"的感觉。

### 协作激活（重要）
- **被邀请方首次进入主页：** 展示**价值，而非任务**——"这是你们的共享计划，你们一起完成了 30%"——绝不是"这里有分配给你的 12 项任务"。
- 给被邀请方**一个微不足道的首个动作**（如：选个主题色、对一个预算数字点反应），让他们先参与再被要求干活。

---

## 完成标准
- [ ] 主页一屏内显示：倒计时 + 最多 3 个下一步 + 温和进度
- [ ] 所有文案是完成正向的，没有任何亏欠 / 落后 / 红色警报措辞
- [ ] 被邀请方首次进入看到的是"共享进度"而非"分配给你的任务"
- [ ] 主页有且只有一个主要操作方向（做下一件事）

---

## 不做（护栏）
- 不做信息仪表盘式的密集数据墙
- 不做进度条 / 百分比制造的压力
- 不加探索 / 推荐 / 内容流
