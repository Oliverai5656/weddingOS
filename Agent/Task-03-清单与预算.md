# Task 03 — 清单 + 预算

> 依赖：Task 00, 01
> 这是两个核心工作面。清单解决"顺序焦虑"，预算解决"超支恐惧"。

---

## 目标
让用户能管理任务（知道何时做什么）和钱（计划 vs 实际，含红包收入），过程简单不焦虑。

---

## 范围 A：清单 Checklist

- **按时间轴分组：** `现在 / 即将 / 以后`（now / soon / later）。
- **渐进式披露：** 突出"现在"组，"即将 / 以后"折叠或弱化。不要一次甩 80 个任务。
- **仪式切换：** 多仪式时，可切换查看某场仪式的任务，或看全部。
- **操作：** 勾选完成、添加 / 编辑任务、设 due 时间。
- 任务可选关联 ceremony_id。
- 完成任务时给温和的正向反馈，不做夸张庆祝动画。

### 清单完成标准
- [ ] 任务正确按 now/soon/later 分组并可视
- [ ] 多仪式可切换 / 筛选
- [ ] 勾选、增删改任务都能跑通且双人同步
- [ ] 默认折叠未来任务，避免信息过载

---

## 范围 B：预算 Budget

- **分类列表：** 每类显示 planned vs actual。
- **总额对比：** **先显示"在预算内 / 还剩 RM X"**，不要先甩红色超支数字。
- **红包 / ang pow 收入追踪（SEA 必需）：** 预算项支持 income 类型，单独汇总收入。
- **净额视图：** 支出总额 - 红包收入 = 实际净花费（可选展示）。
- **操作：** 增删改分类、填 planned / actual 金额。

### 预算完成标准
- [ ] 每个分类显示计划 vs 实际
- [ ] 总览先以正向 / 中性措辞呈现（剩余额度），超支才低调提示
- [ ] 红包收入能记录并单独汇总
- [ ] 数字变化双人同步

---

## 不做（护栏）
- 不做预算预测 / 优化模型
- 不做礼物清单 registry
- 预算不用红色警报轰炸用户
- 清单不做游戏化 / 连续打卡
