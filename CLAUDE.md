# CLAUDE.md — 婚礼规划 Web App 构建总调度

> 角色：你是这个项目的**编排者 / orchestrator**。
> 按顺序派发 Task 00 → 04。每个任务完成并通过"完成标准"后，才进入下一个。
> 不要跳过依赖。不要自行添加未列出的功能。

---

## 项目背景（必读，每个 Task 都遵守）

- **是什么：** 给一对新人用的婚礼规划网页 App。核心价值 = 两个人看到同一份计划，永远知道"下一步做什么"，过程平静不焦虑。
- **平台：** 网页版（手机浏览器 + 电脑浏览器），响应式。**不做原生 App。**
- **市场：** 马来西亚 / 东南亚。数据模型从一开始就支持**多场仪式**、**红包收入追踪**、**宾客关联多仪式**。**不要用西方单场婚礼默认模型。**
- **用途：** 内部自用工具，无变现需求。不要加任何付费墙、广告、推销。

---

## 全局约束（所有 Task 强制）

- **屏幕数量：** 第一阶段封顶 6 屏（引导、主页、清单、预算、宾客、设置）。
- **导航：** 最多 4 个目的地（今天、清单、预算、宾客）。设置放角落图标。不加第五个 tab。
- **每屏一个主要操作。**
- **编辑冲突：** last-write-wins，不做协同算法。
- **通知 / 文案：** 完成正向（"这周完成了 3 件事"），**绝不**亏欠式（"还剩 12 项，进度落后"）。
- **视觉：** 柔和配色、大留白、排版主导、安静紧迫感。无红色警报、无游戏化、无进度条压力、无暗黑模式。

---

## 绝不做（Non-Goals — 不要"好心"加上）

商家平台 / 供应商目录、婚礼网站生成器、电子请柬 / 在线 RSVP、礼物清单 registry、原生 App、AI 聊天机器人、家人 / 宾客登录、社交流。

每个 Task 完成时，回答三个问题：
1. 是否只做了本 Task 范围内的事？（没有偷偷扩张）
2. 是否遵守了全局约束（屏幕数、文案语气、视觉）？

三个都"是"才算完成。
